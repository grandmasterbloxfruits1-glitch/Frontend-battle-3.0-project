=== FEATURE 1: Matrix-Driven Pricing & Performance-Isolated Currency Switcher ===

- A currency switcher: INR (₹), USD ($), EUR (€)
- 3 pricing tiers (e.g. Starter / Pro / Enterprise)

Pricing MUST be computed from a multi-dimensional config object structured exactly like this (values are placeholders, pick reasonable real numbers):

const pricingMatrix = {
  tiers: [
    { id: 'starter', name: 'Starter', baseMonthlyRateUSD: 19 },
    { id: 'pro', name: 'Pro', baseMonthlyRateUSD: 49 },
    { id: 'enterprise', name: 'Enterprise', baseMonthlyRateUSD: 99 }
  ],
  currencies: {
    USD: { symbol: '$', exchangeRate: 1,    regionalTariff: 1.00 },

    EUR: { symbol: '€', exchangeRate: 0.92, regionalTariff: 1.03 }
  },
  annualDiscountMultiplier: 0.8  // flat 20% discount, applied only when billing = annual
};

finalPrice = baseMonthlyRateUSD * exchangeRate * regionalTariff * (billingCycle === 'annual' ? annualDiscountMultiplier : 1)

=== FEATURE 2: Bento-to-Accordion Wrapper with State Persistence ===
A single feature-showcase component, same underlying data (5 feature items), that displays differently by breakpoint:
- Desktop (>768px): CSS grid Bento layout (mixed card sizes look intentional, not a uniform grid).
- Mobile (≤768px): the same items become a vertical Accordion list — each item is a <button> header + a collapsible panel.

Behavior requirements:
- Use matchMedia('(max-width: 768px)') with an addEventListener('change', ...) listener to detect breakpoint crossing — do NOT use a generic resize listener firing on every pixel.
- On desktop, track which bento card is currently hovered/focused in a single JS variable (e.g. activeFeatureIndex), updated on pointerenter/pointerleave (and focus/blur for keyboard users).
- Context Lock: if the user is hovering a bento card and resizes the window past the mobile breakpoint, the matchMedia change handler must check activeFeatureIndex and, if set, programmatically expand that exact item's accordion panel — and it must expand WITH the smooth transition already defined (not snap open instantly).
- Accordion open/close animation: use CSS grid-template-rows transition from 0fr to 1fr (with overflow:hidden on the panel wrapper) — this keeps panel content in the DOM/accessible tree at all times (required for SEO crawlability), it just visually collapses.
- aria-expanded on each accordion trigger button, aria-controls pointing to its panel id, panel has role="region".
- If no card is hovered when the breakpoint is crossed, leave all accordion panels collapsed by default.
- This entire feature must be hand-written vanilla JS/CSS — zero animation or accordion libraries.

MOTION TIMING (apply consistently everywhere):
- Hover/toggle micro-interactions: 150–200ms, ease-out curve.
- Structural layout reflows (accordion expand/collapse, bento-to-accordion transition): 300–400ms, ease-in-out curve.
